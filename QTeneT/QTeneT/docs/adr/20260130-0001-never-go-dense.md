# ADR-0001: Never Go Dense

Date: 2026-01-30

## Context
QTT/TT exists to avoid dense materialization at extreme resolutions.
Dense operations are an escape hatch but create catastrophic failure modes
(memory blowups, silent performance collapse).

## Decision
QTeneT adopts **Never Go Dense** as a default product invariant.

- Dense reconstruction is permitted only via explicit API calls labeled as
  escape hatches (e.g., `reconstruct()`), and should be auditable.

## Consequences
- Operator/solver APIs must accept and return QTT/TT-native types.
- Tests must guard against accidental densification.

## Alternatives
- Allow dense fallbacks everywhere (rejected: too easy to regress).
