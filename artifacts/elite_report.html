3. The Solution: QTT-Native Navier-Stokes

HyperTensor's QTT-NS solver operates entirely in compressed tensor-train format. The 3D velocity field is represented as a sequence of small matrices (TT cores), with storage O(r² · 3 · n_bits) — logarithmic in grid size, not cubic.

How It Works

  1. Morton Ordering: The N³ grid is mapped to a 1D vector via Z-order curve, then reshaped as a tensor product of 2×2×2 blocks — one per bit level.
  2. TT Compression: Each field is decomposed into a train of small matrices (cores), connected by bond indices of rank r ≪ N.
  3. Native Operations: Curl, Laplacian, Hadamard products, and inner products all execute directly on TT cores — never decompressing to dense.
  4. Adaptive Truncation: After each operation, a rank-profiled rSVD sweep maintains compression while preserving physics.
No mesh. No dense arrays. No data transfer bottleneck. The solver generates, evolves, and stores the complete 3D flow field in compressed form from start to finish. At 4096³, even geometry initialization is zero-dense (analytical separable masks). 11. QTT Engineering Rules Compliance

All six fundamental QTT engineering rules are verified and enforced in the production solver:

✓1. QTT is Native
Morton-ordered TT cores. Zero external format conversion. All operations execute directly on TT core tensors.
✓2. SVD = rSVD
Randomized SVD with threshold=48 fires on every truncation. 16× speedup per SVD call over full decomposition.
✓3. Python Loops → Triton
Hadamard, inner, and MPO dispatch to @triton.autotune GPU kernels. Auto-selected block sizes.
✓4. Adaptive Rank
Bell-curve rank profile: higher scale → lower rank. Mean rank utilization as low as 13% at 512³.
✓5. No Decompression
Zero dense reconstruction in the solver hot path. All time-stepping operations execute entirely on compressed TT cores.
✓6. No Dense
Separable fields (sponge, corrections) built zero-dense via analytical outer products. Dense init only for geometry SDF.

Appendix A: Solver Architecture & Algorithms

A.1 Solver Data Flow

1. QTT State Vector — u = (u_x, u_y, u_z) in TT format, O(r² · 3n) per field
↓
2. RHS Evaluation — ω = ∇×u (QTT curl), u×ω (shared cross product), ν∇²u (QTT Laplacian)
↓
3. RK2 (Heun) Integration — k1 = rhs(u), k2 = rhs(u + dt·k1), u_new = u + dt/2·(k1 + k2)
↓
4. Brinkman IB — u = u + u ⊙ (mask_impl − 1), correction-based
↓
5. Sponge Layer — u = u + u ⊙ (decay − 1) + complement, correction-based, separable
↓
6. Adaptive Truncation — QR left-sweep → rSVD right-sweep with bell-curve rank profile
↓
7. Energy Safety Valve — If E_new > E_old: rescale by √(E_old/E_new) · 0.999

Appendix B: 4096³ Zero-Dense Mask Construction

B.1 The Problem

At 4096³, the dense body mask array requires 4096³ × 4 bytes = 275 GB — far exceeding the 28 GB system RAM and 8 GB VRAM. The standard dense SDF evaluation is impossible. TCI (Tensor Cross Interpolation) in Morton space also fails: the Ahmed body occupies only 0.18% of the domain, making 1D Morton fibers extremely unlikely to cross the localized 3D geometry.

B.2 Solution: Analytical Separable Decomposition

The Ahmed body bounding box is axis-aligned, allowing a separable factorization:

χ(x,y,z) ≈ χ_x(x) ⊙ χ_y(y) ⊙ χ_z(z)

where each factor is a smooth tanh bump:
  χ_a(a) = 0.5 · (tanh((a − a_min)/ε) − tanh((a − a_max)/ε))
  ε = 2·dx  (2-cell smoothing width)

Each 1D factor is evaluated on the N-point grid (only N = 4096 floats), decomposed into a 1D TT via TT-SVD, then interleaved with identity pass-through cores to create a 3D QTT field via separable_{x,y,z}_field_qtt. The full 3D mask is assembled by two Hadamard products: χ = (χ_x ⊙ χ_y) ⊙ χ_z.

B.3 mask_implicit and brink_corr

The Brinkman penalty mask mask_impl = 1/(1 + c·χ) is not exactly separable. However, the separable approximation mi_x · mi_y · mi_z differs from the true value only in the thin transition layer at the body surface — inside the body (χ≈1) and outside (χ≈0), both forms agree exactly. The Brinkman correction brink_corr = mask_impl − 1 is computed via QTT subtraction.
