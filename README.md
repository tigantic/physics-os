## The Problem

Physics simulation at scale requires supercomputers. A single high-fidelity CFD run at 10вҒ№ grid points consumes terabytes of RAM, days of wall time, and six-figure cloud bills. This gatekeeps the most valuable engineering decisions on Earth вҖ” aircraft design, fusion reactor optimization, drug molecule simulation, surgical planning вҖ” behind institutions that can afford the compute.

## The Solution

**HyperTensor** replaces dense numerical arrays with **Quantized Tensor Trains (QTT)** вҖ” a mathematical compression that reduces memory from O(NВі) to O(log N). A simulation that requires 1 TB of RAM in dense format fits in 64 MB. A grid of **4.4 Г— 10В№ВІ degrees of freedom** runs on a single workstation.

This isn't an approximation. Conservation laws are verified to machine precision (О” < 10вҒ»В№вҒө). Every simulation produces a cryptographic trust certificate вҖ” a Lean 4-verified, Halo2-proven, Ed25519-signed proof that the physics actually ran correctly.

| **Maximum Grid** | ~10вҒ№ points | **4.4 Г— 10В№ВІ points** |
| **Memory Scaling** | O(NВі) | **O(log N)** |
| **Hardware** | HPC cluster | **Single workstation** |
| **Formal Verification** | None | **Lean 4 proofs** |

## The Engine: 471K Lines of Physics

The core physics engine (`tensornet/`, 1,082 files, 471,534 LOC) implements a register-based bytecode VM for QTT computation. Domain specifications compile to an intermediate representation (IR) of QTT operations вҖ” no dense arrays are ever materialized.

### Seven Physics Domains

| Domain | Key | Governing Equation |
|--------|-----|-------------------|
| Viscous Burgers | `burgers` | u_t + uu_x = ОҪu_xx |
| Maxwell TE | `maxwell` | вҲӮ_t E = cвҲӮ_x H, вҲӮ_t H = cвҲӮ_x E |
| Maxwell 3D | `maxwell_3d` | Full curlвҖ“curl system |
| SchrГ¶dinger | `schrodinger` | iв„ҸПҲ_t = -(в„ҸВІ/2m)ПҲ_xx + VПҲ |
| Advection-Diffusion | `advection_diffusion` | u_t + cu_x = Оәu_xx |
| Vlasov-Poisson | `vlasov_poisson` | f_t + vf_x + (q/m)Ef_v = 0 |
| Navier-Stokes 2D | `navier_stokes_2d` | Incompressible vorticity-streamfunction |

### The Never-Dense Guarantee

Every operation stays in TT/QTT format. Dense materialization is architecturally blocked:

- **Rank Governor**: Adaptive bond dimension control after every rank-growing operation
- **Memory**: O(rВІ В· log N) where r is the controlled bond dimension
- **GPU**: Auto-detect CUDA вҶ’ Triton JIT for hot paths вҶ’ graceful CPU fallback
- **Exascale**: Demonstrated at **16,384Ві = 4.4 Г— 10В№ВІ DOF** on commodity hardware
