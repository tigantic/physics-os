### The Fundamental Insight

A genuine photograph or video is the output of a physical process: photons travel from light sources, interact with surfaces (reflection, refraction, scattering, absorption), pass through a lens system (diffraction, chromatic aberration, vignetting), and arrive at a sensor (photoelectric effect, noise statistics, Bayer demosaicing).

Every step obeys Maxwell's equations. A deepfake does not simulate Maxwell's equations. It approximates the visual output using neural networks that have no concept of photon transport, optical path length, Fresnel reflection coefficients, atmospheric Rayleigh/Mie scattering, or sensor quantum efficiency.

**If you can simulate the physics of image formation from first principles, you can test whether a recorded image is consistent with those physics.**

This is not artifact detection. It is physics verification. The generator cannot defeat it because defeating it requires solving Maxwell's equations — which is what HyperTensor does.

## Technical Architecture

### Physics of Image Formation

```
Scene → Light Transport → Lens → Sensor → Image

1. Light Sources
   - Sun/sky: Rayleigh scattering (λ⁻⁴ dependency)
   - Artificial: spectral emission profile
   - Computed via: Maxwell FDTD or ray tracing

2. Surface Interaction
   - Reflection: Fresnel equations (n₁ sin θ₁ = n₂ sin θ₂)
   - Diffuse: Lambertian (I ∝ cos θ)
   - Specular: BRDF (Cook-Torrance, GGX)
   - Computed via: Maxwell for wave effects, geometric optics for rays

3. Atmospheric Propagation
   - Rayleigh scattering: σ ∝ λ⁻⁴ (blue sky)
   - Mie scattering: σ depends on particle size (haze, fog)
   - Absorption: Beer-Lambert law (I = I₀ exp(-αd))
   - Computed via: Navier-Stokes + radiative transfer

4. Lens System
   - Diffraction: Airy pattern, MTF
   - Chromatic aberration: dispersion (n varies with λ)
   - Geometric distortion: barrel, pincushion
   - Computed via: wave optics (Fourier transform of aperture)

5. Sensor
   - Quantum efficiency: η(λ)
   - Shot noise: Poisson statistics
   - Read noise: Gaussian
   - Bayer pattern: color filter array
```

### Consistency Tests (No Deepfake Can Pass All)

| Test | Physics | What It Checks |
|------|---------|---------------|
| Shadow direction consistency | Geometric optics | All shadows point to same light source(s) |
| Shadow softness vs. distance | Penumbra geometry | Soft shadows = extended source, distance-dependent |
| Specular highlight position | Fresnel reflection | Highlight position consistent with surface normal + light |
| Atmospheric color gradient | Rayleigh scattering | Distant objects shift blue, near objects neutral |
| Lens chromatic aberration | Dispersion | Color fringing pattern consistent with real lens |
| Noise statistics | Poisson + Gaussian | Noise distribution matches sensor physics |
| Reflection physics | Fresnel equations | Reflection intensity varies with angle correctly |
| Motion blur direction | Integration over exposure | Blur direction consistent with object motion |
| Depth-of-field | Thin lens equation | Blur circle varies correctly with depth |
| Color temperature | Blackbody radiation | Illuminant color matches physical source |

A deepfake generator would need to simultaneously satisfy ALL of these constraints — which requires solving the forward physics problem. At that point, you're not generating a deepfake; you're doing physics-based rendering, and the proof would verify it as physically consistent (which is correct — it IS physically consistent because it was rendered from physics).

### QTT Formulation

```
For each image/frame:
  1. Estimate scene geometry (depth map, surface normals)
  2. Estimate light sources (position, intensity, spectrum)
  3. Forward-simulate image formation via Maxwell/geometric optics
  4. Compare predicted image vs. actual image
  5. Residual analysis: physics violations localized in pixel space

QTT acceleration:
  - Light field as 4D QTT (2 spatial + 2 angular dimensions)
  - Atmospheric scattering as QTT (smooth → low rank)
  - Lens PSF as QTT (Airy function → compressible)
  - Forward simulation in QTT: O(log N) per pixel evaluation
  
For 4K video (3840 × 2160 × 30fps):
  Dense: ~750 MB/second
  QTT: ~5 MB/second (rank ≤ 16 for natural scenes)
  Real-time viable on GPU
```
