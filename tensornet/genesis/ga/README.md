# QTT-GA: Geometric Algebra with QTT Compression

**Layer 26 of TENSOR GENESIS Protocol**

## Overview

Geometric Algebra (GA), also known as Clifford Algebra, provides a unified mathematical framework for geometry, physics, and computer science. This module implements GA operations with QTT compression, enabling computations on exponentially large multivector spaces.

## Mathematical Foundation

### Clifford Algebra Cl(p,q,r)

A Clifford algebra is generated by an orthonormal basis {eвӮҒ, ..., eвӮҷ} with:
- **p** basis vectors squaring to +1 (positive signature)
- **q** basis vectors squaring to -1 (negative signature)  
- **r** basis vectors squaring to 0 (degenerate)

The algebra has dimension 2вҒҝ where n = p + q + r.

### Common Algebras

| Algebra | p | q | r | Dimension | Application |
|---------|---|---|---|-----------|-------------|
| Complex | 0 | 1 | 0 | 2 | Complex numbers |
| Quaternions | 0 | 2 | 0 | 4 | 3D rotations |
| VGA (3D) | 3 | 0 | 0 | 8 | Vector geometry |
| PGA (3D) | 3 | 0 | 1 | 16 | Projective geometry |
| CGA (3D) | 4 | 1 | 0 | 32 | Conformal geometry |
| Spacetime | 1 | 3 | 0 | 16 | Special relativity |

### The Geometric Product

The geometric product is the fundamental operation:

```
ab = aВ·b + aвҲ§b
```

Where:
- **aВ·b** is the inner (dot) product (grade lowering)
- **aвҲ§b** is the outer (wedge) product (grade raising)

### Multivector Structure

A general multivector has 2вҒҝ components organized by grade:
- **Grade 0**: Scalars (1 component)
- **Grade 1**: Vectors (n components)
- **Grade 2**: Bivectors (n(n-1)/2 components) вҖ” oriented areas
- **Grade 3**: Trivectors вҖ” oriented volumes
- **Grade n**: Pseudoscalar (1 component)

### Rotors

Rotors represent rotations without gimbal lock:

```
R = exp(-ОёB/2)
```

Where B is a unit bivector (rotation plane) and Оё is the angle.

Rotation of vector v:
```
v' = Rv RМғ   (sandwich product)
```

Rotors compose naturally: RвӮҒвӮӮ = RвӮӮRвӮҒ

## QTT Innovation

### The Key Insight

A multivector in Cl(n) has 2вҒҝ components. For large n, this is impossible to store classically:
- n=10: 1,024 components вң“
- n=20: 1,048,576 components вҡ пёҸ
- n=40: 1,099,511,627,776 components вң—

**QTT Representation**: Since 2вҒҝ = 2Г—2Г—...Г—2 (n times), multivectors naturally map to QTT format with n cores of size (rвӮ–вӮӢвӮҒ, 2, rвӮ–).

### Complexity Comparison

| Operation | Classical | QTT |
|-----------|-----------|-----|
| Storage | O(2вҒҝ) | O(nВ·rВІ) |
| Geometric product | O(2ВІвҒҝ) | O(nВ·rвҒ¶) |
| Grade projection | O(2вҒҝ) | O(nВ·rВі) |
| Rotor application | O(2ВІвҒҝ) | O(nВ·rвҒ¶) |

### Sparse Structure

Many GA operations produce sparse results:
- Projecting to a single grade
- Products of low-grade elements
- Physical multivectors (often just vectors + bivectors)

QTT exploits this sparsity with low TT ranks.

## Usage Examples

### Basic Multivector Operations

```python
from tensornet.genesis.ga import CliffordAlgebra, vector, geometric_product

# Create 3D VGA
vga = CliffordAlgebra(p=3, q=0, r=0)

# Create vectors
a = vector(vga, [1, 0, 0])  # e1
b = vector(vga, [0, 1, 0])  # e2

# Geometric product
ab = geometric_product(a, b)  # e1e2 (bivector)

# Inner and outer products
from tensornet.genesis.ga import inner_product, outer_product
dot = inner_product(a, b)    # 0 (orthogonal)
wedge = outer_product(a, b)  # e12
```

### Rotations with Rotors

```python
from tensornet.genesis.ga import rotor_from_angle_plane, apply_rotor, bivector
import math

# Create rotation in xy-plane
B = bivector(vga, {(0, 1): 1.0})  # e12
R = rotor_from_angle_plane(math.pi/4, B)  # 45-degree rotor

# Rotate a vector
v = vector(vga, [1, 0, 0])
v_rotated = apply_rotor(R, v)
```

### Conformal Geometric Algebra

```python
from tensornet.genesis.ga import (
    ConformalGA, point_to_cga, cga_line, cga_sphere, meet
)

cga = ConformalGA()

# Create geometric objects
p1 = point_to_cga(cga, [0, 0, 0])
p2 = point_to_cga(cga, [1, 0, 0])
line = cga_line(cga, p1, p2)

sphere = cga_sphere(cga, center=[0, 0, 0], radius=1.0)

# Intersection
intersection = meet(line, sphere)
```

### QTT-Compressed Multivectors

```python
from tensornet.genesis.ga import QTTMultivector, qtt_geometric_product

# Create QTT multivector for large algebra
qtt_mv = QTTMultivector.random(n_generators=40, rank=10)

# Geometric product in QTT format
result = qtt_geometric_product(qtt_mv, qtt_mv)
```

## Theoretical Background

### Basis Blades

For n generators, the 2вҒҝ basis blades are indexed by subsets of {1,...,n}:
- вҲ… вҶ’ 1 (scalar)
- {1} вҶ’ eвӮҒ
- {2} вҶ’ eвӮӮ
- {1,2} вҶ’ eвӮҒвӮӮ
- etc.

Binary encoding: blade index k has basis vectors where bit i is set.

### Cayley Table

The geometric product of basis blades follows sign rules from the metric:
```

eбөўeбөў = gбөўбөў    (from metric signature)
```

### Exponential Map

For bivector B with BВІ = -|B|ВІ:
```

```
This is the rotor formula.
